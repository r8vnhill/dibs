### Comparación con C#

En **C#**, al igual que en Kotlin, existen colecciones mutables que permiten agregar, eliminar y modificar elementos después de ser creados. C# proporciona una amplia variedad de colecciones mutables, como `List`, `HashSet` y `Dictionary`, las cuales son comparables con `MutableList`, `MutableSet`, y `MutableMap` en Kotlin. A continuación, veremos una comparación de cómo funcionan estas colecciones mutables en ambos lenguajes.

#### Listas Mutables

En **C#**, la clase `List<T>` se utiliza para almacenar una lista mutable de elementos. Al igual que en Kotlin, permite agregar, eliminar y modificar elementos.

```csharp showLineNumbers
var mutableList = new List<string> { "C#", "Java" };
mutableList.Add("Kotlin");
mutableList[1] = "Swift";
Console.WriteLine(string.Join(", ", mutableList));  // Output: C#, Swift, Kotlin
```

En **Kotlin**, el equivalente sería `MutableList`.

```kotlin showLineNumbers
val mutableList: MutableList<String> = mutableListOf("Kotlin", "Java")
mutableList.add("Scala")
mutableList[1] = "Groovy"
println(mutableList)  // Output: [Kotlin, Groovy, Scala]
```

#### Sets Mutables

En **C#**, el `HashSet<T>` es un conjunto mutable que no permite elementos duplicados, al igual que `MutableSet` en Kotlin.

```csharp showLineNumbers
var mutableSet = new HashSet<string> { "Kotlin", "Java" };
mutableSet.Add("Scala");
mutableSet.Remove("Java");
Console.WriteLine(string.Join(", ", mutableSet));  // Output: Kotlin, Scala
```

En **Kotlin**, el uso de `MutableSet` es similar:

```kotlin showLineNumbers
val mutableSet: MutableSet<String> = mutableSetOf("Kotlin", "Java")
mutableSet.add("Scala")
mutableSet.remove("Java")
println(mutableSet)  // Output: [Kotlin, Scala]
```

#### Diccionarios Mutables

En **C#**, el `Dictionary<TKey, TValue>` es el equivalente mutable de `MutableMap` en Kotlin. Permite asociar claves con valores, y las entradas se pueden agregar, eliminar o modificar.

```csharp showLineNumbers
var mutableMap = new Dictionary<string, int> {
    { "C#", 1 },
    { "Java", 2 }
};
mutableMap["Scala"] = 3;
mutableMap.Remove("Java");
Console.WriteLine(string.Join(", ", mutableMap));  // Output: [C#=1, Scala=3]
```

En **Kotlin**, el `MutableMap` se comporta de manera muy similar:

```kotlin showLineNumbers
val mutableMap: MutableMap<String, Int> = mutableMapOf("Kotlin" to 1, "Java" to 2)
mutableMap["Scala"] = 3
mutableMap.remove("Java")
println(mutableMap)  // Output: {Kotlin=1, Scala=3}
```

### Similitudes entre Kotlin y C#

1. **Modificación en tiempo de ejecución**: Ambas lenguas ofrecen colecciones mutables que permiten agregar, eliminar y modificar elementos después de que la colección ha sido creada.
2. **Operaciones comunes**: Tanto en C# como en Kotlin, las colecciones mutables proporcionan operaciones como `add`, `remove`, `clear`, entre otras.
3. **Flexibilidad**: En ambos lenguajes, las colecciones mutables permiten la manipulación dinámica de datos, adaptándose a diferentes escenarios en tiempo de ejecución.

### Diferencias entre Kotlin y C#

1. **Sintaxis**: Aunque las operaciones son similares, la sintaxis de Kotlin tiende a ser más concisa, mientras que C# es más explícito en la declaración de colecciones.
2. **Nombres de clases**: En C#, las clases como `List`, `HashSet`, y `Dictionary` son ampliamente utilizadas, mientras que en Kotlin se prefieren `MutableList`, `MutableSet`, y `MutableMap`.
3. **Inmutabilidad por defecto**: En Kotlin, las colecciones son inmutables por defecto, y para obtener una colección mutable, se debe utilizar las versiones `Mutable`. En C#, la mayoría de las colecciones son mutables por defecto.
