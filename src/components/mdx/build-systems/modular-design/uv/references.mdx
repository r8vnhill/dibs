## 📖 Referencias

### 🔥 Recomendadas

- [📄 *"Managing Python workspaces with uv"*](https://federico.is/posts/2024/12/18/managing-python-workspaces-with-uv/) por **Federico Ragona**: Una guía clara y práctica sobre cómo estructurar proyectos Python en múltiples módulos usando `uv`. Explica cómo declarar workspaces, gestionar dependencias locales y organizar el código de forma escalable. Relevante porque refuerza el enfoque modular y profesional que buscamos al separar la lógica de negocio de la lógica de aplicación.

### 🔹 Adicionales

- [🌐 "Using workspaces"](https://docs.astral.sh/uv/concepts/projects/workspaces/) de **la documentación oficial de _uv_**: Describe cómo organizar múltiples módulos en un solo repositorio, compartir un `lockfile` y definir dependencias entre paquetes locales. Relevante para esta lección porque respalda la estructura modular adoptada, permitiendo que `core` y `app` colaboren sin necesidad de publicación externa ni entornos virtuales separados.
