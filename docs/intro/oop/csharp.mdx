## C#

### Singleton

En **C#**, no existe la palabra clave `object` para crear un singleton de manera directa, pero el patrón **singleton** se puede implementar utilizando una clase estática con un constructor privado, lo que garantiza que solo se cree una instancia de la clase.

```csharp
public class MySingleton {
    private static readonly MySingleton instance = new MySingleton();

    private MySingleton() {}

    public static MySingleton Instance => instance;

    public string Name => "Singleton";

    public string Greet() => $"Hello, {Name}";
}
```

#### Similitudes

- Ambos lenguajes soportan el patrón singleton.
- En ambos casos, se garantiza que solo haya una instancia del objeto.

#### Diferencias

- Kotlin tiene soporte nativo para singletons mediante `object`, lo que simplifica su creación.
- En C#, se requiere una implementación manual utilizando una clase con un constructor privado.

### Clases

Las clases en _C#_ son similares, pero con sintaxis y convenciones propias del lenguaje.

```csharp
public class Person {
    public string Name { get; }

    public Person(string name) {
        Name = name;
    }

    public string Greet() => $"Hello, {Name}";
}
```

#### Similitudes

- Ambos lenguajes definen clases de forma similar, permitiendo almacenar propiedades y comportamientos.
- Soportan propiedades inmutables, en Kotlin usando `val` y en C# usando propiedades solo de lectura (`get`).

#### Diferencias

- En Kotlin, los constructores primarios se definen en la cabecera de la clase, mientras que en C# se declaran dentro del cuerpo de la clase.

### Interfaces

Tanto Kotlin como C# soportan **interfaces**, que definen contratos que las clases deben implementar.

```csharp
public interface IGreeter {
    string Greet();
}

public class Person : IGreeter {
    public string Greet() => $"Hello, {Name}";
}
```

#### Similitudes

- Ambos lenguajes permiten definir interfaces con métodos abstractos y métodos con implementación predeterminada.
- Las clases pueden implementar múltiples interfaces en ambos lenguajes.

:::info
En _C#_, la convención es que los nombres de las interfaces comiencen con `I`, como `IGreeter` en lugar de `Greeter`.
:::

### Clases Abstractas

Las **clases abstractas** en Kotlin y C# son similares en cuanto a su propósito: no se pueden instanciar directamente y pueden contener tanto métodos abstractos como implementados.

```csharp
public abstract class Animal {
    public abstract string Sound();
    public string Walk() => "Walking...";
}

public class Dog : Animal {
    public override string Sound() => "Woof";
}
```

#### Similitudes

- Ambos lenguajes permiten la creación de clases abstractas con métodos abstractos y concretos.
- Tanto en Kotlin como en C#, una clase abstracta puede ser extendida por otras clases que implementen sus métodos abstractos.

#### Diferencias

- No hay grandes diferencias conceptuales, aunque la sintaxis varía entre los dos lenguajes.

### Encapsulación

En C#, el concepto es similar, utilizando `public`, `private`, `protected`, e `internal`.

```csharp
public class Person {
    private int Age = 30;

    public int GetAge() => Age;
}
```

#### Similitudes

- Ambos lenguajes permiten controlar la visibilidad de los miembros de una clase utilizando modificadores de acceso.

#### Diferencias

- En C#, `internal` se usa para limitar la visibilidad al ensamblado, mientras que en Kotlin `internal` restringe la visibilidad al módulo.

### Constructores Primarios y Secundarios

En **C#**, no existe el concepto explícito de constructores primarios como en Kotlin, pero es posible emular este comportamiento mediante la sobrecarga de constructores.

```csharp
public class Person {
    public string Name { get; }
    public int Age { get; }

    // Constructor principal
    public Person(string name) {
        Name = name;
        Age = 0;
    }

    // Constructor secundario
    public Person(string name, int age) : this(name) {
        Age = age;
    }
}
```

En este ejemplo, el constructor principal inicializa solo el nombre, mientras que el constructor secundario delega parte de la inicialización al principal y también inicializa la edad.

#### Similitudes

- Ambos lenguajes permiten sobrecargar constructores para ofrecer diferentes formas de inicializar una clase.
- Los constructores en ambos lenguajes pueden delegar parte de la lógica de inicialización a otros constructores.

#### Diferencias

- Kotlin tiene una sintaxis más concisa con su constructor primario definido en la cabecera de la clase, y utiliza el bloque `init` para inicializaciones adicionales.
- En C#, todos los constructores se definen dentro del cuerpo de la clase, lo que puede requerir más código.
