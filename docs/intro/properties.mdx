Las **propiedades** son fundamentales para definir el estado de una clase y funcionan como variables miembro. En Kotlin, las propiedades combinan el comportamiento de los campos con los métodos *getter* y *setter*, ofreciendo una manera más concisa y expresiva de manejar datos.

Kotlin genera automáticamente un método `get` para propiedades inmutables (declaradas con `val`), y tanto `get` como `set` para propiedades mutables (declaradas con `var`). No obstante, es posible personalizar estos métodos si se requiere un comportamiento más específico.

### Ejemplo de Propiedad con Getter y Setter Personalizados en Kotlin

```kotlin
class GettersAndSetters {
    var name: String = "John"
        get() { // Getter personalizado
            return field
        }
        set(value) { // Setter personalizado
            field = value
        }
}
```

En **Kotlin**, `field` es una palabra clave especial que representa el valor almacenado internamente en la propiedad. Es útil en *getters* y *setters* personalizados para acceder o modificar ese valor.

### Propiedad Calculada

Una **propiedad calculada** es una propiedad cuyo valor se evalúa cada vez que se accede a ella. En Kotlin, puedes usar un *getter* para hacer esto:

```kotlin
class GettersAndSetters {
    val now: String
        get() = Clock.System.now().toString()
}
```

Cada vez que se accede a la propiedad `now`, se genera un valor basado en la hora actual.

### Propiedad con Setter Privado

A veces, necesitas permitir que otras partes del programa lean el valor de una propiedad, pero restringir quién puede modificarla. En Kotlin, esto se logra mediante un *setter* privado:

```kotlin
class GettersAndSetters(age: Int) {
    var age: Int = age
        private set // Setter privado
}
```

### Propiedad con Backing Field

En Kotlin, el **backing field** permite controlar el acceso a una propiedad, manteniendo el valor almacenado en una variable privada:

```kotlin
class GettersAndSetters(private var _age: Int) {
    val age: Int
        get() = _age
}
```

## Ejemplos en otros lenguajes
<details>
    <summary>Ver ejemplos en otros lenguajes</summary>

    ### Java

    En **Java**, puedes usar la biblioteca **Lombok** para generar automáticamente getters y setters sin necesidad de escribirlos manualmente. Esto hace que el código sea más conciso, similar a Kotlin:

    ```java
    import lombok.Getter;
    import lombok.Setter;

    public class GettersAndSetters {
        @Getter @Setter
        private String name = "John";
    }
    ```

    ### TypeScript

    En **TypeScript**, las propiedades también pueden tener *getters* y *setters* personalizados para controlar el acceso a los valores:

    ```typescript
    class GettersAndSetters {
        private _name: string = 'John';

        get name(): string { // Getter personalizado
            return this._name;
        }

        set name(value: string) { // Setter personalizado
            this._name = value;
        }
    }
    ```

    ### Python

    En **Python**, las propiedades se pueden definir con el decorador `@property`, proporcionando getters y setters personalizados:

    ```python
    class GettersAndSetters:
        def __init__(self):
            self._name = "John"

        @property
        def name(self):  # Getter personalizado
            return self._name

        @name.setter
        def name(self, value):  # Setter personalizado
            self._name = value
    ```

    ### C#

    En **C#**, las propiedades combinan `get` y `set` en una única estructura, lo que facilita su uso:

    ```csharp
    public class GettersAndSetters {
        private string _name = "John";

        public string Name { // Propiedad con getter y setter
            get { return _name; }
            set { _name = value; }
        }
    }
    ```

    ### Swift

    En **Swift**, las propiedades también permiten personalizar los accesores mediante `get` y `set`:

    ```swift
    class GettersAndSetters {
        private var _name: String = "John"

        var name: String {
            get { return _name } // Getter personalizado
            set { _name = newValue } // Setter personalizado
        }
    }
    ```
</details>

## En resumen

Las **propiedades** permiten gestionar el estado de una clase de manera eficiente al combinar el comportamiento de los campos con los métodos *getter* y *setter*. En **Kotlin**, estos métodos se generan automáticamente, pero pueden ser personalizados para un mayor control. Además, el uso de **propiedades calculadas**, **setters privados**, y **backing fields** proporciona flexibilidad en la forma en que se manejan y protegen los datos. Los ejemplos en otros lenguajes como **Java**, **TypeScript**, **Python**, **C#**, y **Swift** muestran que este concepto es ampliamente utilizado y personalizable.
