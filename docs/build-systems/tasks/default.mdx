Gradle ofrece un conjunto de tareas predefinidas que facilitan la automatización de las operaciones más comunes en los
proyectos de software. Estas tareas son estándares y se pueden ejecutar en cualquier proyecto Gradle sin necesidad de
configuración adicional.

## Tarea `build`

La tarea `build` es una de las más usadas en Gradle. Se encarga de ejecutar una serie de subtareas que incluyen la
compilación del código fuente, la ejecución de pruebas y la generación de artefactos (como un JAR). Al ejecutar:

```bash
./gradlew build
```

Gradle realizará todas las operaciones necesarias para compilar y construir el proyecto.

## Tarea `clean`

La tarea `clean` elimina todos los archivos generados durante la compilación, dejando el proyecto en un estado limpio.
Esto es útil cuando deseas hacer una compilación completa desde cero. Al ejecutar:

```bash
./gradlew clean
```

Gradle eliminará los directorios de salida como `build/` y otros archivos generados.

## Tarea `assemble`

La tarea `assemble` compila y empaqueta la aplicación, pero no ejecuta las pruebas. Esto es útil cuando solo necesitas
generar los artefactos de salida. Se ejecuta con:

```bash
./gradlew assemble
```

Gradle compilará y empaquetará el proyecto sin ejecutar pruebas.

## Tarea `check`

La tarea `check` ejecuta todas las pruebas y verificaciones de calidad del código. Incluye pruebas unitarias, análisis
estático de código y otras validaciones que el proyecto tenga definidas. Para ejecutar esta tarea, utiliza:

```bash
./gradlew check
```

Gradle verificará que todo el código cumple con las pruebas y las reglas de calidad.

## Configuración de Tareas por Defecto

Gradle permite personalizar tareas predefinidas para ajustarlas a las necesidades del proyecto. Por ejemplo, la tarea
`test` se puede configurar para especificar el framework de pruebas, los eventos a registrar y otros detalles.

```kotlin title="convention-plugins/src/main/kotlin/jvm.conventions.gradle.kts"
tasks.test {
    useJUnitPlatform()
    testLogging {
        events("passed", "skipped", "failed")
        showStandardStreams = true
    }
}
```

En este caso, se configura la tarea `test` para usar JUnit como framework de pruebas y registrar eventos de pruebas
pasadas, omitidas y fallidas.

## Ejemplos en Otras Plataformas

Aunque este curso está centrado en Gradle, otras herramientas de construcción y plataformas fuera de la JVM tienen
tareas similares para manejar la automatización de procesos.

### Ejemplos en Herramientas Fuera de la JVM

- **npm (JavaScript/Node.js)**:
    - `npm run build`: Compila y empaqueta un proyecto JavaScript.
    - `npm run clean`: Limpia los archivos generados durante la compilación.
    - `npm test`: Ejecuta las pruebas definidas en el proyecto.

- **Make (C/C++ y otras plataformas)**:
    - `make all`: Compila y enlaza todos los archivos necesarios para construir el proyecto.
    - `make clean`: Elimina archivos temporales y binarios generados durante la compilación.

- **Cargo (Rust)**:
    - `cargo build`: Compila el código y genera los binarios del proyecto.
    - `cargo clean`: Limpia los archivos generados durante la compilación.
    - `cargo test`: Ejecuta las pruebas unitarias y de integración.

### Ejemplos en Herramientas JVM

Aunque Gradle es muy popular en la JVM, también existen otras herramientas como Maven y SBT que siguen patrones
similares:

- **Maven**:
    - `mvn package`: Equivalente a `gradle build`.
    - `mvn clean`: Equivalente a `gradle clean`.
    - `mvn test`: Ejecuta las pruebas unitarias.

- **SBT (Scala)**:
    - `sbt compile`: Compila el proyecto Scala.
    - `sbt clean`: Limpia los archivos generados durante la compilación.
    - `sbt test`: Ejecuta las pruebas unitarias de Scala.

## Resumen

Las tareas predefinidas en Gradle son esenciales para automatizar los procesos clave en un proyecto. Ya sea para
compilar, probar o empaquetar una aplicación, Gradle tiene tareas que cubren la mayoría de las necesidades de un
proyecto típico. Además, al personalizar estas tareas, puedes ajustarlas a las particularidades de tu flujo de
trabajo, lo que proporciona gran flexibilidad.

Y aunque Gradle es una herramienta poderosa en el ecosistema JVM, otros sistemas de construcción como npm, Cargo o Make, también tienen capacidades similares para automatizar procesos en sus respectivas plataformas.
