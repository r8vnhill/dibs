## JavaScript/TypeScript: Jest

La composición de **matchers** también es una característica común en frameworks de testing como **Jest**, utilizado en proyectos **JavaScript** y **TypeScript**. Al igual que en Kotlin con Kotest, Jest permite combinar múltiples validaciones para crear pruebas más expresivas y robustas.

1. **Composición de Matchers**: Al igual que en Kotest, en Jest puedes combinar varios matchers utilizando operaciones lógicas como `and` o `or`. Esto te permite realizar validaciones más complejas en una única prueba.
    ```typescript
    expect(password).toMatch(/\d/) &&
        expect(password).toMatch(/[A-Z]/) &&
        expect(password).toMatch(/[a-z]/) &&
        expect(password.length).toBeGreaterThanOrEqual(16);
    ```
2. **Expresividad**: Ambos frameworks permiten escribir validaciones complejas de manera declarativa y legible, facilitando la creación de tests comprensibles y fáciles de mantener.
3. **Matchers Reutilizables**: Tanto en Kotest como en Jest, puedes definir **matchers reutilizables** que encapsulan la lógica de validación. Esto es útil cuando necesitas validar condiciones complejas en múltiples pruebas.
