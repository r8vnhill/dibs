Los **Matchers** son una forma de expresar de manera más declarativa las expectativas sobre el comportamiento del código en pruebas unitarias. Estas funciones permiten verificar si un valor cumple con una condición específica y, en caso contrario, lanzan una excepción. Los **Matchers** mejoran la legibilidad y consistencia en el código, ofreciendo mensajes de error detallados que facilitan la identificación y corrección de errores.

### Beneficios de los Matchers

<ProCons>
    <Pros>
        - **Declarativos y Expresivos**: Hacen que las pruebas sean más legibles, ya que expresan las expectativas de manera más natural.
        - **Mensajes de Error Informativos**: Proporcionan mensajes detallados que ayudan a diagnosticar fácilmente por qué falló una prueba.
        - **Estandarización**: Mejoran la consistencia al escribir pruebas en toda la base de código.
        - **Amplio Soporte**: Son compatibles con diversos frameworks de testing como **AssertJ** (Java), **NUnit** (C#), **Scalatest** (Scala), **Jest** (TypeScript), **PyHamcrest** (Python), **RSpec** (Ruby), **GoogleTest** (C++), y **XCTest** (Swift), entre otros.        
    </Pros>
</ProCons>

### Comparación: Aserciones vs. Matchers

| **Aserciones**                                                                                  | **Matchers**                                                                                                          |
|-------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------|
| Verificaciones básicas que comparan valores y lanzan una excepción si la condición no se cumple | Encapsulan las comparaciones y mensajes de error, proporcionando una API más limpia y expresiva para escribir pruebas |
| `assertNotEquals(user.username, null)`                                                          | `user.username.shouldNotBeNull()`                                                                                     |

Los **Matchers** proporcionan una API más fluida, lo que hace que las pruebas sean más fáciles de leer y escribir, en comparación con las aserciones tradicionales que pueden ser más verbosas.
