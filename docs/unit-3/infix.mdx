Las **funciones infijas** son una forma elegante de llamar a funciones de manera más concisa y fluida. En lugar de usar el formato habitual con puntos y paréntesis, las funciones infijas permiten escribir código más legible y natural, especialmente cuando solo hay un argumento involucrado. Sin embargo, las funciones infijas tienen ciertos requisitos que deben cumplirse para poder usarse de esta forma.

## Requisitos para Definir Funciones Infijas

Para que una función sea declarada como infija, debe cumplir con las siguientes condiciones:

- Debe ser una **función miembro** de una clase o una **función de extensión**.
- Solo puede aceptar **un único parámetro**.
- Debe estar marcada con el modificador `infix`.

Estas reglas aseguran que el uso de funciones infijas sea claro y no cause confusión, dado que su sintaxis es más relajada.

## Ejemplo de Función Infija en una Clase

```kotlin
class Point(val x: Int, val y: Int) {
    infix fun shift(dx: Int): Point = Point(x + dx, y)
}
```

En este ejemplo, `shift` es una función infija que permite mover un punto en el eje `x` por una cantidad `dx`. La sintaxis infija hace que el código sea más legible, simulando una especie de "operación" entre el objeto `Point` y el desplazamiento `dx`.

```kotlin
fun main() {
    val point = Point(1, 2)
    val newPoint = point shift 3
    println("El punto se ha desplazado a x=${newPoint.x}, y=${newPoint.y}")
}
```

Aquí, la función `shift` se invoca de manera infija, es decir, **sin paréntesis** ni punto, lo que le da al código una sintaxis más natural.

## Ejemplo de Función Infija como Extensión

No solo las funciones miembro pueden ser infijas. También es posible declarar **funciones de extensión** como infijas, lo que agrega aún más flexibilidad al lenguaje.

```kotlin
class Point(val x: Int, val y: Int)

infix fun Point.shift(dx: Int): Point = Point(x + dx, y)
```

En este caso, la función `shift` es una extensión de la clase `Point`, permitiendo usar la misma sintaxis infija con objetos de la clase.

```kotlin
fun main() {
    val point = Point(1, 2)
    val newPoint = point shift 3
    println("El punto se ha desplazado a x=${newPoint.x}, y=${newPoint.y}")
}
```

Al igual que en el ejemplo anterior, se utiliza la función `shift` sin necesidad de paréntesis ni punto, lo que simplifica el código y lo hace más intuitivo.

## Consideraciones

Las funciones infijas pueden mejorar significativamente la legibilidad del código, pero es importante usarlas en situaciones donde realmente aporten claridad. Además, dado que una función infija puede hacer que el código se parezca a una operación nativa, es esencial que el propósito y la funcionalidad de la función sean claros y consistentes con las expectativas del usuario.

Si bien las funciones infijas ofrecen una sintaxis más natural, es crucial tener en cuenta las **limitaciones** y **contexto** en el que se usan para garantizar que el código siga siendo mantenible y fácil de entender para todxs lxs desarrolladorxs que lo lean o lo mantengan en el futuro.
