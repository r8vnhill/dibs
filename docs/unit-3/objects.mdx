## Objetos en Kotlin

En Kotlin, la palabra clave `object` se utiliza para declarar un **singleton**, que es un objeto único que existe durante toda la ejecución del programa. Este patrón es especialmente útil cuando se necesita una única instancia global de una clase para coordinar acciones o gestionar estados a lo largo del sistema. En lugar de crear múltiples instancias de una clase, el patrón singleton garantiza que solo haya una, lo que facilita la centralización de ciertos comportamientos o datos.

### Ejemplo de Singleton

El siguiente ejemplo muestra cómo declarar un objeto singleton en Kotlin. En este caso, `Dice` es un objeto que simula un dado de 6 caras. El objeto tiene una propiedad `sides` que define el número de caras y un método `roll()` que genera un número aleatorio entre 1 y 6.

```kotlin
object Dice {
    val sides = 6

    fun roll() = (1..sides).random()
}
```

En este ejemplo:
- **`object Dice`**: Declara un objeto llamado `Dice`. Este objeto es un singleton, lo que significa que solo hay una instancia de `Dice` en todo el programa.
- **`val sides`**: Es una propiedad constante que define cuántas caras tiene el dado.
- **`fun roll()`**: Este método simula el lanzamiento del dado, generando un número aleatorio entre 1 y 6.

Este patrón es útil para evitar la creación de múltiples instancias innecesarias y para tener un punto centralizado de control.

:::tip Ejercicio: Implementación de un Gestor de Eventos
<Tabs>
    <TabItem value="Ejercicio" label="Ejercicio">
        El objetivo de este ejercicio es que desarrolles un objeto llamado `EventManager` que actúe como un gestor para administrar una lista de eventos. Cada evento estará representado como una cadena de texto (`String`), y el gestor será responsable de almacenar y devolver los eventos registrados.

        **Requisitos**:
        1. **Estructura de Datos**:
        - Utiliza un `MutableList<String>` para almacenar los eventos. Esta lista será mutable, lo que significa que podrás agregar eventos a ella.
        - Inicializa la lista utilizando `mutableListOf()`.

        2. **Métodos Requeridos**:
        - **Agregar Evento**: Define un método llamado `addEvent(event: String)` que permita añadir un nuevo evento a la lista.
        - **Obtener Eventos**: Define un método `getEvents(): List<String>` que retorne una copia inmutable de la lista de eventos. Para lograr esto, puedes utilizar el método `toList()`, que genera una copia de la lista mutable, protegiendo así la integridad de los datos.

        **Ejemplo de uso**:
        ```kotlin
        fun main() {
            EventManager.addEvent("Concierto de Rock")
            EventManager.addEvent("Festival de Cine")

            println(EventManager.getEvents())
            // Debería imprimir: ["Concierto de Rock", "Festival de Cine"]
        }
        ```

        **Puntos Clave**:
        - **Centralización del Control**: El uso de un singleton como `EventManager` centraliza la gestión de eventos, asegurando que siempre haya una
        única instancia manejando los datos.
        - **Copia Inmutable**: El método `getEvents()` retorna una copia de la lista de eventos, evitando modificaciones accidentales desde el exterior.

        Este ejercicio te ayudará a comprender cómo manejar datos compartidos de manera segura y eficiente utilizando objetos singleton en Kotlin.
    </TabItem>
    <TabItem value="Solución" label="Solución">
        ```kotlin
        object EventManager {
            val events = mutableListOf<String>()

            fun addEvent(event: String) {
                events.add(event)
            }

            fun getEvents(): List<String> {
                return events.toList()
            }
        }
        ```
