
## Try/Result

La mónada Try (o Result en algunos lenguajes) se utiliza para manejar operaciones que pueden fallar, encapsulando tanto el éxito como el fallo de una operación.
Componentes Principales:
Success (Ok en algunos lenguajes):
Representa un resultado exitoso de la operación.
Contiene el valor resultante de la operación.
Failure (Err en algunos lenguajes):
Representa un fallo o error durante la operación.
Contiene la excepción o el mensaje de error que describe el problema.
Motivación:
Evita el uso de excepciones no controladas.
Facilita el manejo explícito de errores en lugar de depender de excepciones lanzadas.
Hace que el flujo de control sea más claro y seguro al obligar al programador a considerar tanto el éxito como el fallo.
