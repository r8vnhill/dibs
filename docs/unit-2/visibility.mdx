El **encapsulamiento** es una técnica fundamental en la programación orientada a objetos que consiste en ocultar los detalles internos de un objeto. Los datos de un objeto solo pueden ser accedidos o modificados a través de métodos definidos específicamente para este propósito. Esto garantiza que los datos del objeto se manipulen de manera controlada, evitando errores o comportamientos inesperados al restringir el acceso directo a sus componentes.

El encapsulamiento también ayuda a proteger la integridad de los datos, ya que se asegura que cualquier cambio en el estado del objeto esté controlado por la lógica definida en sus métodos.

## Modificadores de Visibilidad

En Kotlin (y otros lenguajes orientados a objetos), los **modificadores de visibilidad** determinan qué partes del código pueden acceder a los miembros de una clase. Aquí hay una descripción de los modificadores más comunes:

| Modificador | Visibilidad                                                                                                             |
|-------------|-------------------------------------------------------------------------------------------------------------------------|
| `public`    | Accesible desde cualquier parte del código. Es el valor por defecto si no se especifica un modificador.                 |
| `private`   | Accesible solo dentro de la clase que lo contiene. Protege los detalles internos de la clase.                           |
| `protected` | Accesible desde la clase que lo contiene y sus subclases. Limita el acceso fuera de la jerarquía de herencia.           |
| `internal`  | Accesible solo desde el módulo donde está definida la clase. Esto es útil para limitar el acceso dentro de un proyecto. |

### Ejemplo de Encapsulamiento en Kotlin

Consideremos un ejemplo en Kotlin donde se utiliza el modificador `private` para proteger un atributo sensible como la contraseña de un usuario. Solo un método público puede cambiar esta propiedad, pero bajo ciertas condiciones.

```kotlin
class User {
    private var password: String = "123456"

    fun updatePassword(newPassword: String) {
        if (newPassword.length >= 6) {
            password = newPassword
        } else {
            println("La contraseña debe tener al menos 6 caracteres")
        }
    }
}
```

En este caso:
- La propiedad `password` es privada, lo que impide que se acceda directamente desde fuera de la clase.
- El método `updatePassword()` es público, pero valida el nuevo valor antes de actualizar la propiedad, asegurando que el cambio cumpla con ciertos criterios.

Este ejemplo demuestra cómo el encapsulamiento ayuda a proteger el estado de un objeto y a garantizar que las modificaciones solo se realicen de manera controlada.

De esta manera, el uso adecuado de los modificadores de visibilidad ayuda a mantener un diseño de software más robusto y seguro, mejorando la mantenibilidad y evitando accesos no deseados a los componentes internos de una clase.
