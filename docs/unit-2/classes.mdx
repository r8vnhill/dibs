### Clases en Kotlin

En la programación orientada a objetos, una **clase** es un modelo o plantilla que define las características y comportamientos de los objetos. En otras palabras, es la estructura que utilizamos para crear instancias (objetos) con propiedades y métodos específicos. En Kotlin, las clases son fundamentales para organizar y reutilizar código de manera eficiente.

#### Definición de una Clase

Consideremos el siguiente ejemplo de una clase `Player` en el contexto de un juego. Esta clase representa a un jugador y define dos propiedades básicas: el nombre del jugador y sus puntos de vida.

```kotlin
class Player {
    var name: String = ""
    var lifePoints: Int = 8000
}
```

En este código:
- **`class Player`**: Declara una clase llamada `Player`.
- **`var name: String`**: Define una propiedad mutable `name` de tipo `String`, que almacena el nombre del jugador.
- **`var lifePoints: Int`**: Define una propiedad mutable `lifePoints` de tipo `Int`, que almacena los puntos de vida del jugador. Por defecto, se le asigna el valor de `8000`.

#### Uso de Clases: Instanciación

Una vez que has definido la clase, puedes crear **instancias** de esta clase, es decir, objetos. A cada instancia de `Player` se le pueden asignar valores específicos para sus propiedades.

El siguiente código muestra cómo crear un objeto de tipo `Player`, asignar valores a sus propiedades y acceder a ellas para realizar operaciones, como imprimir el estado actual del jugador:

```kotlin
fun main() {
    val player = Player()  // Creación de un objeto de tipo Player
    player.name = "Seto Jaiva"  // Asignación de nombre al jugador
    player.lifePoints = 7000  // Asignación de puntos de vida

    // Impresión del estado del jugador
    println("El jugador ${player.name} tiene ${player.lifePoints} puntos de vida")
}
```

En este ejemplo:
- **`val player = Player()`**: Se crea una nueva instancia de `Player` usando el constructor predeterminado de Kotlin. En este caso, no estamos pasando parámetros al constructor, ya que no hemos definido uno personalizado.
- **`player.name = "Seto Jaiva"`**: Asignamos el valor `"Seto Jaiva"` a la propiedad `name` del jugador.
- **`player.lifePoints = 7000`**: Cambiamos los puntos de vida del jugador a `7000`.
- **Interpolación de cadenas**: Utilizamos `${}` para insertar las propiedades `name` y `lifePoints` dentro de la cadena de texto que imprimimos con `println()`.

#### Ventajas del Uso de Clases

1. **Reutilización de código**: Una clase es un componente reutilizable. Puedes crear múltiples instancias con las mismas características pero con diferentes valores para las propiedades.
2. **Encapsulamiento**: Las clases permiten encapsular el estado y el comportamiento, haciendo que el código sea más modular y más fácil de mantener.
3. **Abstracción**: A través de las clases, puedes modelar entidades del mundo real en el código de forma clara y precisa, ocultando los detalles de implementación y exponiendo solo lo necesario.

#### Propiedades Mutables e Inmutables

En el ejemplo anterior, utilizamos `var` para definir propiedades mutables, es decir, que pueden cambiar a lo largo del ciclo de vida del objeto. Si una propiedad no debería cambiar una vez asignada, podemos usar `val` para hacerla inmutable:

```kotlin
class Player(val name: String) {
    var lifePoints: Int = 8000
}
```

En este caso, la propiedad `name` es inmutable, por lo que no se puede cambiar después de crear el objeto `Player`, mientras que `lifePoints` sigue siendo mutable.
