## üîÅ Declaraci√≥n `for`

<Important>
    En Kotlin, `for` es una **declaraci√≥n**, no una expresi√≥n.
</Important>

Esto significa que **no produce un valor**: se utiliza para **ejecutar acciones** (como imprimir o modificar datos), pero **no puede formar parte de una expresi√≥n mayor** ni asignarse a una variable.

<details>
    <summary>
        ¬øY en otros lenguajes?
    </summary>

    A diferencia de Kotlin, donde `for` se limita a ejecutar efectos, lenguajes como **Scala**, **Haskell** o **Python** permiten recorrer estructuras de forma **expresiva** y, al mismo tiempo, **producir un resultado**.

    #### Scala ‚Äì `for` como expresi√≥n

    En Scala, los bucles `for` pueden generar nuevas colecciones mediante `yield`:

    ```scala showLineNumbers title="Bucle for en Scala"

        yield x * 2
    // Resultado: List(2, 4, 6)
    ```

    Este patr√≥n se conoce como **for-comprehension**, y combina operaciones como `map`, `flatMap` y `filter` de forma declarativa.

    #### Haskell ‚Äì `forM` mon√°dico

    Aunque Haskell no tiene un `for` tradicional, puedes recorrer estructuras con efectos usando `forM` del m√≥dulo `Control.Monad`:

    ```haskell showLineNumbers title="Bucle forM en Haskell"

    forM [1..3] (\x -> return (x * 2))
    -- Resultado (en una m√≥nada pura): [2, 4, 6]
    ```

    `forM` es una variante de `mapM` con los argumentos invertidos, lo que mejora la legibilidad en algunos contextos.

    #### Python ‚Äì Comprensiones de listas

    Python permite construir listas de manera compacta a trav√©s de *list comprehensions*:

    ```python showLineNumbers title="Comprensi√≥n de listas en Python"

    # Resultado: [2, 4, 6]
    ```

    Tambi√©n existen variantes para conjuntos y diccionarios.

    ---

    En todos estos lenguajes, el bucle puede **producir un valor**, lo que permite **encadenar transformaciones sin efectos secundarios**.

    En Kotlin, ese papel lo cumplen funciones como `map`, `filter` o `fold`, mientras que `for` est√° reservado para acciones con efectos colaterales (como imprimir o modificar variables).
</details>

```kotlin title="Sintaxis b√°sica de un bucle for"

    // cuerpo del ciclo

```

Las llaves `{}` son opcionales cuando el cuerpo del ciclo contiene **una sola instrucci√≥n**, pero se recomienda **usarlas siempre** por motivos de **legibilidad** y para evitar errores al agregar nuevas l√≠neas m√°s adelante.

### üîÅ Ejemplos de bucles `for`


<Explanation>
    - `1..5`: Crea un rango **cerrado**, que incluye el 5.
    - `5 downTo 1`: Cuenta en reversa, creando un rango **descendente**.
    - `step`: Define el tama√±o del paso (por ejemplo, de 2 en 2).
    - `..<` y `until`: Crean rangos **semiabiertos**, que **excluyen** el l√≠mite superior.

Tambi√©n puedes usar rangos con otros tipos, como caracteres:


:::tip Rangos m√°s all√° de n√∫meros

En Kotlin, los rangos no se limitan a enteros: puedes usarlos con **cualquier tipo que implemente [`Comparable`](https://kotlinlang.org/api/core/kotlin-stdlib/kotlin/-comparable/)**, como `Char`, `String` o `Double`.

Esto permite escribir expresiones como:


:::

::::warning ¬øSe pueden recorrer con `for`?

No siempre.  
Aunque muchos tipos pueden formar un rango, **no todos los rangos pueden usarse en un ciclo `for`**.

Solo ciertos tipos como `Int`, `Long` y `Char` implementan lo necesario para que un rango sea tambi√©n una **secuencia iterable**.  
En cambio, rangos como `1.0..5.0` (`Double`) o `"Foo".."Foz"` (`String`) **no pueden recorrerse directamente con `for`**, porque Kotlin **no sabe c√≥mo generar el siguiente valor autom√°ticamente**.

:::note ¬øQu√© valor sigue?

Pi√©nsalo un momento:

- ¬øQu√© n√∫mero sigue despu√©s de `1.0`?  
  ¬ø`1.1`, `1.01`, `1.000001`, `1 + 1e-100`‚Ä¶?
- ¬øY qu√© `String` viene despu√©s de `"Foo"`?  
  ¬ø`"Fop"`?, ¬ø`"Fooa"`?, ¬ø`"Foofighters"`?

Estas preguntas **no tienen una √∫nica respuesta correcta**, por lo que Kotlin prefiere **no asumir una l√≥gica arbitraria**.

:::

Por eso es importante distinguir entre **crear un rango** y **recorrer un rango**:  
**no todos los rangos son secuencias.**

::::

## üìè Verificar si un valor pertenece a un rango

## üîÑ Declaraci√≥n `while`

En Kotlin, la declaraci√≥n `while` permite repetir un bloque de c√≥digo **mientras se cumpla una condici√≥n booleana**.  
Existen dos variantes principales:

<Box>
    ```kotlin title="Bucle while"

        // cuerpo del ciclo
    }
    ```

    - Verifica la condici√≥n **antes de cada iteraci√≥n**.
    - Si la condici√≥n es falsa **desde el principio**, el cuerpo **no se ejecuta ni una sola vez**.
    - √ötil cuando **no sabes cu√°ntas veces** se repetir√° el ciclo, pero necesitas comprobar algo antes de empezar.
</Box>
<Box>
    ```kotlin title="Bucle do-while"

        // cuerpo del ciclo
    } while (condici√≥n)
    ```

    - Ejecuta el cuerpo del ciclo **una vez antes de verificar la condici√≥n**.
    - Luego repite mientras la condici√≥n siga siendo verdadera.
    - √ötil cuando **la acci√≥n debe realizarse al menos una vez**, como al pedir una entrada de usuario o realizar una verificaci√≥n inicial.

## üéØ Conclusiones

Los ciclos son esenciales para controlar el flujo de un programa, y Kotlin los presenta con un enfoque que prioriza la **claridad, seguridad y expresividad**.  
En esta lecci√≥n recorrimos distintas formas de repetici√≥n ‚Äî`for`, `while`, `do-while` y `repeat`‚Äî y aprendimos a elegir la m√°s adecuada seg√∫n el contexto.

Tambi√©n profundizamos en conceptos fundamentales como la diferencia entre **rangos** y **secuencias recorribles**, lo que permite evitar errores comunes al iterar sobre estructuras complejas o poco convencionales.

### üîë Puntos clave

- `for` es una **declaraci√≥n**, √∫til para ejecutar efectos, pero **no produce valores**.
- Los rangos de tipos como `Int`, `Long` o `Char` pueden recorrerse directamente; otros como `Double` o `String` no.
- El operador `in` permite verificar pertenencia en rangos y colecciones, y puede usarse en condiciones o en expresiones `when`.
- `while` y `do-while` ofrecen control basado en condiciones, con una diferencia clave en el momento en que se eval√∫an.
- `repeat` es una funci√≥n expresiva para repetir un bloque de c√≥digo un n√∫mero fijo de veces.

### üß∞ ¬øQu√© nos llevamos?

M√°s que aprender la sintaxis de los bucles, esta lecci√≥n nos invit√≥ a **reflexionar sobre el dise√±o del c√≥digo**.  
No se trata solo de repetir acciones, sino de **comunicar claramente la intenci√≥n**: cu√°ndo iterar, c√≥mo hacerlo y con qu√© estructura lograr el mejor equilibrio entre legibilidad y funcionalidad.

Este enfoque es fundamental en el dise√±o de bibliotecas: cada decisi√≥n debe favorecer un c√≥digo que sea **legible, predecible y reutilizable**.

Finalmente, escribir buenos ciclos no es una habilidad aislada, sino parte de un ciclo mayor: el de aprender, aplicar, refinar‚Ä¶ y volver a aprender.  
Como en la programaci√≥n, el valor no est√° en repetir por repetir, sino en **repetir con prop√≥sito**.
