El manejo de entrada de usuario es esencial en programas interactivos. Kotlin ofrece métodos seguros y convenientes para leer datos desde la consola, adaptándose a diversas necesidades de procesamiento de entrada.

## Funciones para Lectura de Entrada en Kotlin

Para leer la entrada de usuario en Kotlin, se puede utilizar la función **`readlnOrNull()`**. Esta función es segura y devuelve un valor de tipo `String?`, que puede ser `null` si no hay más datos disponibles o si el usuario presiona Enter sin ingresar nada. Esta función permite manejar los casos de valores nulos de forma segura, evitando excepciones no deseadas y facilitando un código más robusto.

:::danger Función Deprecada en Versiones Recientes
Kotlin también ofrece la función **`readln()`**, aunque esta función fue comúnmente utilizada en versiones anteriores de Kotlin, ha sido deprecada debido a que lanza una excepción si no hay más entrada disponible. Esto puede interrumpir inesperadamente el flujo del programa, haciendo su uso menos seguro y predecible. Por esta razón, es preferible utilizar `readlnOrNull()`.
:::

## Ejemplo de Uso de `readlnOrNull()`

El siguiente ejemplo muestra cómo leer nombres de usuario de manera segura utilizando `readlnOrNull()`. El programa termina cuando el usuario presiona Enter sin ingresar ningún dato:

```kotlin
fun main() {
    println("Enter names. Press only Enter to finish.")
    while (true) {
        println("Enter a name:")
        val input = readlnOrNull()
        if (input.isNullOrEmpty()) {
            println("No name entered. Exiting the program.")
            break  // Se interrumpe el ciclo al no haber más entrada
        } else {
            println("Name entered: $input")
        }
    }
}
```

### Explicación del Código

- **Ciclo Infinito**: Utilizamos `while (true)` para solicitar al usuario la entrada de manera continua hasta que decida no ingresar más datos.
- **Lectura Segura**: `readlnOrNull()` lee la entrada del usuario. Si el usuario no ingresa nada y presiona Enter, el valor devuelto será `null` o una cadena vacía.
- **Manejo de Entrada Vacía**: Si la entrada es nula o vacía, el programa imprime un mensaje indicando que no se ingresó ningún nombre y termina el ciclo usando `break`.
- **Manejo de Entrada Válida**: Si el usuario ingresa un nombre válido, el programa lo imprime y sigue pidiendo más nombres hasta que se interrumpa el ciclo.
